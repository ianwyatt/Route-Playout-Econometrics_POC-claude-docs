## Notes

### Key Performance Metrics (Production)
- **Database**: 1.28B playout records, 252.7M audience records cached
- **Cache Coverage**: 826 campaigns × 69 days (Aug 6 - Oct 13, 2025)
- **Speedup**: 2,400x - 8,500x vs. direct API calls
- **Campaign Browser**: 838 campaigns, <100ms query time

### Critical Lessons Learned
1. **Never use correlated subqueries on tables >1M records** - Use window functions instead
2. **Always check `pg_stat_activity` before migrations** - Kill duplicate processes
3. **Use `EXPLAIN ANALYZE`** for query planning on large datasets
4. **Window functions > Correlated subqueries** - 1000x+ speedup
